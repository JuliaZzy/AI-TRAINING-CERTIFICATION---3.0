## 题号 3.1.2-1：用户使用习惯与系统性能分析报告

### 一、用户使用习惯：不同时段的亮度与颜色偏好

#### 1.1 时段划分与数据分布

本数据集涵盖当日 07 时至 23 时共 17 个小时，每小时约有 58—60 条记录，数据分布较为均匀，能够较好地反映全天各时段的用户行为规律。

**各时段平均亮度与色温汇总：**

| 时段 | 时间区间 | 平均亮度（0-100） | 平均色温（K） | 主导场景 |
|------|---------|----------------|------------|--------|
| 早晨 | 07:00—09:59 | **51.7** | 3,696 | Work Mode（45次） |
| 上午 | 10:00—12:59 | 51.2 | 3,662 | Reading Mode（56次） |
| 午后 | 13:00—14:59 | 49.4 | 3,689 | Reading Mode（31次） |
| 下午 | 15:00—17:59 | 50.3 | **3,797** | Reading Mode（53次） |
| 傍晚 | 18:00—20:59 | 48.2 | 3,726 | Relax Mode（65次） |
| 夜晚 | 21:00—23:59 | **47.9** | 3,596 | Sleep Mode（52次） |

#### 1.2 亮度偏好分析

从亮度变化规律来看，用户的亮度需求随一天时间呈现出**"早高晚低"**的总体趋势：

- **早晨（07—09 时）平均亮度最高，达 51.7**，其中 09 时单小时均值高达 56.0，为全天峰值。这与用户早晨起床后需要较强光线以快速进入清醒状态的生理需求高度吻合。
- **上午（10—12 时）亮度维持在 51.2** 的较高水平，用户处于工作或学习的活跃状态，对明亮环境的需求持续。
- **午后至下午（13—17 时）**亮度小幅回落至 49—51 区间，整体平稳，午后短暂的休息需求使亮度略有下调。
- **傍晚（18—20 时）亮度开始明显下降，均值为 48.2**，用户由工作模式逐渐切换为休闲放松模式，对强光的需求减少。
- **夜晚（21—23 时）平均亮度降至全天最低值 47.9**，22 时单小时均值仅为 43.9，Sleep Mode 成为该时段最频繁使用的场景，反映用户有意识地降低亮度以营造睡眠准备环境。

**亮度分布特征：** 全天亮度中位数为 50，标准差为 29.6，说明用户亮度调节范围跨度大，个体差异显著，系统需要具备宽泛的亮度自适应能力。

#### 1.3 色温偏好分析

色温的变化规律相对复杂，呈现出**"午后偏暖、傍晚先升后降、夜间偏低"**的特征：

- **早晨（07—09 时）色温约为 3,696 K**，属于中性白光区间，有助于用户在清晨保持清醒。
- **下午（15—17 时）色温达到 3,797 K**（其中 17 时均值为 3,930 K），是全天最高时段；19 时更达到全天单小时峰值 4,006 K（偏冷白光），可能与部分用户在傍晚加班或进行需要专注的活动相关。
- **夜晚（21—23 时）色温下降明显，23 时均值仅 3,339 K**，为全天最低，偏暖黄色调，有助于减少蓝光对睡眠的干扰，体现用户的健康照明意识。
- **Work Mode 场景色温均值最高（3,842 K）**，符合冷白光提升专注度的使用逻辑；Sleep Mode 与 Relax Mode 色温均在 3,640—3,655 K 之间，偏向暖光以营造舒适放松的氛围。

#### 1.4 各场景亮度与色温特征对比

| 场景 | 使用次数 | 占比 | 平均亮度 | 平均色温（K） |
|------|--------|------|--------|------------|
| Relax Mode（放松模式） | 273 | 27.3% | 50.3 | 3,640 |
| Reading Mode（阅读模式） | 264 | 26.4% | 49.8 | 3,656 |
| Work Mode（工作模式） | 237 | 23.7% | **50.7** | **3,842** |
| Sleep Mode（睡眠模式） | 226 | 22.6% | **48.0** | 3,652 |

- **Work Mode** 亮度和色温均为最高，与提升工作效率的照明需求一致；
- **Sleep Mode** 亮度最低，有助于促进褪黑素分泌，辅助睡眠；
- **Relax Mode** 和 **Reading Mode** 参数接近，偏中性白光，亮度适中。

#### 2.1 整体场景使用频率

从全天 1,000 条记录来看，四大智能场景的使用次数分布相对均衡，但仍存在明显的差异：

```

Reading Mode（阅读模式）：264 次 █████████████████████████   26.4%

Sleep Mode（睡眠模式）： 226 次 █████████████████████       22.6%
```

**Relax Mode 使用频率最高（27.3%）**，略领先于其他三种模式；**Sleep Mode 使用频率最低（22.6%）**，差距约 4.7 个百分点。整体来看，四种场景使用较为均衡，没有极端的冷热差异。

#### 2.2 各时段场景使用偏好

| 时段 | 最常用场景 | 次数 | 次常用场景 | 最少用场景 |
|------|---------|-----|---------|---------|
| 早晨（07—09时） | Work Mode | 45 | Reading Mode | Relax Mode（36） |
| 上午（10—12时） | Reading Mode | 56 | Relax Mode | Work Mode（36） |
| 午后（13—14时） | Reading Mode | 31 | Relax Mode | Sleep Mode（21） |
| 下午（15—17时） | Reading Mode | 53 | Work Mode | Sleep Mode（34） |
| 傍晚（18—20时） | **Relax Mode** | **65** | Reading Mode | Sleep Mode（36） |
| 夜晚（21—23时） | **Sleep Mode** | **52** | Relax Mode | Reading Mode（40） |

**关键发现：**

1. **早晨（07—09 时）Work Mode 最受欢迎（45 次）**：用户起床后倾向于快速进入工作或学习状态，照明系统对此时段的工作场景切换响应需求较高。

2. **上午至下午 Reading Mode 持续占主导**：上午（56 次）和下午（53 次）均为最高，体现出白天用户有大量阅读、学习需求，Reading Mode 是全天使用时长最稳定的场景。

3. **傍晚（18—20 时）Relax Mode 使用次数全天最高，达 65 次**：这是 Relax Mode 的黄金时段，用户在结束一天工作后需要放松，系统应在此时段主动推荐或自动切换至 Relax Mode。

4. **夜晚（21—23 时）Sleep Mode 跃升为主导场景（52 次）**：与傍晚的 Relax Mode 形成清晰的"放松—入睡"转换链条，说明用户有较强的睡眠准备意识。

5. **Sleep Mode 全天整体使用频率最低（22.6%）**：仅在夜晚时段表现突出，其他时段使用明显偏少，存在进一步挖掘场景价值的空间（如午休模式）。

#### 2.3 场景使用规律总结

- **频繁使用（推荐重点优化）**：Relax Mode（全天均衡+傍晚高峰）、Reading Mode（白天主力场景）
- **使用适中**：Work Mode（早晨和上午集中使用）
- **使用较少（存在拓展潜力）**：Sleep Mode（仅夜晚集中，日间午休场景待开发）

#### 3.5 延迟瓶颈综合判断

综合以上分析，系统响应延迟的主要瓶颈体现在三个维度：

1. **场景级瓶颈**：Relax Mode 高延迟次数最多，该场景可能涉及更复杂的光效过渡算法或设备联动，导致处理耗时偏长；
2. **时段级瓶颈**：上午（10—12 时）为日间使用高峰，并发请求量大，服务器或网关处理能力受到考验；
3. **系统级瓶颈**：约 27.5% 的请求超过 1.5 秒，且极值接近 2.0 秒，揭示系统存在**周期性性能抖动**（可能源于内存回收、网络波动或固件限制），需从底层架构层面进行优化。

## 题号 3.1.2-2：智能照明系统优化方向与解决方案

基于上述分析，提出以下三个核心优化方向：

---

### 优化方向一：构建时段自适应照明推荐引擎，提升用户使用体验

**问题背景：**

数据显示用户的亮度与色温偏好具有显著的时间规律性：早晨偏好高亮度（均值 51.7）、夜晚偏好低亮度暖光（22 时均值仅 43.9，色温降至 3,339 K）；傍晚（18—20 时）Relax Mode 使用次数全天最高（65 次），夜晚 Sleep Mode 成为主导场景。然而目前这些场景均需用户手动触发，缺乏智能预判与主动推送机制。

**解决方案：**

1. **建立用户照明行为模型**：利用历史数据挖掘每位用户在不同时段的亮度、色温与场景偏好，为每个用户生成个性化的"照明时刻表"，实现"千人千面"的照明策略。

2. **开发时段自动切换功能**：根据时间规律，在关键节点（如 18:00 自动切换至 Relax Mode、21:00 自动切换至 Sleep Mode 准备状态）向用户推送建议或自动执行切换，减少手动操作频次。

3. **引入渐进式亮度调节**：夜间逐步降低亮度（如每 30 分钟降低 5 个单位）并自动收缩色温至暖光区间（< 3,500 K），模拟自然光变化规律，辅助用户建立健康的昼夜节律。

4. **拓展 Sleep Mode 的午休场景**：数据显示 Sleep Mode 在日间使用率极低，可推出"午休模式"子场景（13:00—14:00 时段），结合轻柔降亮与暖光设置，挖掘 Sleep Mode 在非夜间时段的使用价值。

**预期效果：** 减少用户手动操作次数约 30%—40%，提升场景触发的及时性与准确率，增强系统"懂用户"的智能感知。

---

### 优化方向二：针对高延迟场景实施分级优化，将系统响应时间压缩至 1.0 秒以内

**问题背景：**

当前系统平均响应时间为 1.064 秒，P90 达 1.804 秒，P99 接近 2.0 秒，27.5% 的请求响应超过 1.5 秒，用户体验存在明显延迟感。Relax Mode 是高延迟次数最多的场景（36 次高延迟），上午（10—12 时）是响应时间最长的时段（均值 1.125 s）。智能照明的核心体验是"即指即变"，超过 1 秒的响应延迟会显著降低用户满意度。

**解决方案：**

1. **优化 Relax Mode 光效算法**：对 Relax Mode 的场景渲染逻辑进行代码层优化，采用异步预加载机制，在用户触发指令前预先完成色温过渡参数计算，将 Relax Mode 平均响应时间目标压缩至 0.8 秒以内。

2. **部署本地边缘计算网关**：在家庭局域网内部署照明控制边缘节点，将常用指令（场景切换、亮度调节）的处理逻辑下沉至本地执行，避免每次操作都依赖云端往返通信，可将响应时间减少 40%—60%。

3. **引入指令预测与预执行机制**：结合用户行为规律（如傍晚大概率切换 Relax Mode），在预测窗口期（切换前 10—15 分钟）预热相关设备参数，实现"零感知延迟"的场景切换。

4. **建立响应时间分级 SLA 监控机制**：设置实时告警阈值（P50 < 0.8s，P90 < 1.5s，P99 < 1.8s），对持续超标的设备或时段触发自动诊断和降级保护策略，防止长尾延迟扩散。

5. **高峰时段动态扩容**：针对上午（10—12 时）的并发高峰，实施弹性资源调度策略，在预测到高并发前提前扩展服务处理能力，平抑峰值期响应时间波动。

**预期效果：** 将整体平均响应时间从 1.064 秒降低至 0.8 秒以下，P90 响应时间从 1.804 秒降低至 1.2 秒以内，高延迟（>1.5s）请求占比从 27.5% 降低至 10% 以下。

---

### 优化方向三：强化多场景智能联动与个性化推荐，提升场景使用均衡度

**问题背景：**

数据显示四大场景使用频率存在分化：Relax Mode（27.3%）和 Reading Mode（26.4%）占据主导，而 Work Mode（23.7%）和 Sleep Mode（22.6%）使用相对较少。Sleep Mode 在非夜间时段几乎处于"闲置"状态，场景价值未被充分挖掘。此外，各场景之间缺乏智能联动逻辑，用户需要频繁手动切换，降低了系统的智能化体验。

**解决方案：**

1. **构建场景联动逻辑链**：基于数据发现的"Work Mode（早晨）→ Reading Mode（白天）→ Relax Mode（傍晚）→ Sleep Mode（夜晚）"自然使用链条，设计场景自动流转规则，支持用户配置个性化的"一天照明计划"，系统按计划自动执行场景切换。

2. **引入多设备联动协同**：将智能照明与其他智能家居设备（如智能窗帘、空调、音乐）进行场景联动，例如触发 Relax Mode 时同步降低窗帘透光率并播放轻音乐，打造沉浸式场景体验，提升场景黏性和使用频率。

3. **开发使用频率可视化反馈面板**：在 APP 中为用户展示各场景的使用统计（频率、时长、亮度/色温分布），结合健康照明建议（如"您今天夜晚使用高色温照明超过 2 小时，建议切换至暖光模式"），引导用户优化照明习惯，间接提升低频场景的使用率。

4. **针对 Work Mode 和 Sleep Mode 进行场景细分**：
   - **Work Mode** 可细分为"高强度专注（高亮度 + 高色温）"与"轻度工作（中亮度 + 中色温）"两档，满足不同工作强度的需求；
   - **Sleep Mode** 增加"午休模式"与"深度睡眠模式"，前者适用于 13:00—14:00 短暂休息，后者在 22:00 后自动触发并逐步关闭全部灯光，丰富场景覆盖面。

5. **建立个性化场景推荐算法**：利用协同过滤算法，基于相似用户的行为模式向当前用户推荐其尚未使用或使用较少的场景组合，逐步提升系统的场景多样性使用率，使四大场景的使用频率趋于更均衡的分布（目标：各场景偏差不超过 3%）。

**预期效果：** 提升场景切换的智能化程度，减少用户手动干预；Sleep Mode 日间使用率提升 50% 以上；四大场景使用均衡度显著改善，用户整体满意度提升。
