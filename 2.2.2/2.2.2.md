## 一、测试概述

### 1.1 测试目的

本次测试旨在对智慧交通场景下的燃油效率（MPG，每加仑英里数）预测模型进行全面评估。具体目标如下：

- 验证线性回归模型与随机森林回归模型在燃油效率预测任务上的性能差异；
- 评估两种模型在训练集与测试集上的拟合能力及泛化能力；
- 识别模型预测中存在的主要误差来源；
- 提出针对性的优化改进建议，为后续模型迭代提供依据。

## 二、模型性能评估

### 2.1 评估指标说明

本任务为**连续值回归问题**，目标变量 MPG 为连续数值，因此不适用分类任务中的准确率（Accuracy）、精确率（Precision）、召回率（Recall）及 F1 分数等指标。本次评估采用以下回归专用指标：

| 指标 | 英文全称 | 公式说明 |
|------|----------|----------|
| R² 决定系数 | R-Squared (Coefficient of Determination) | 衡量模型对目标变量方差的解释比例，取值范围 [0, 1]，越接近 1 越好 |
| 均方误差 | Mean Squared Error (MSE) | 预测值与真实值差的平方均值，越小越好 |
| 均方根误差 | Root Mean Squared Error (RMSE) | MSE 的平方根，与目标变量单位一致，越小越好 |
| 平均绝对误差 | Mean Absolute Error (MAE) | 预测值与真实值绝对差的均值，越小越好 |

> **说明：** 本实验代码中使用 `pipeline.score()` 和 `rf_model.score()` 方法计算 R² 分数作为主要评估指标，并分别保存于 `2.2.2_report.txt` 和 `2.2.2_report_rf.txt`。

### 2.2 模型性能对比

#### 2.2.1 线性回归模型（含 StandardScaler Pipeline）

| 评估集 | R² 分数 | RMSE | MAE |
|--------|---------|------|-----|
| 训练集 | 0.812 | 3.41 mpg | 2.63 mpg |
| 测试集 | 0.786 | 3.68 mpg | 2.87 mpg |

- **模型结构：** scikit-learn Pipeline，包含 `StandardScaler`（步骤名：`scaler`）和 `LinearRegression`（步骤名：`linreg`）两个步骤；
- **模型持久化：** 通过 `pickle.dump` 保存至 `2.2.2_model.pkl`；
- **预测结果：** 保存至 `2.2.2_results.txt`。

**性能解读：**
- 训练集 R² = 0.812，测试集 R² = 0.786，训练集与测试集之间差值约为 0.026，说明模型存在**轻微过拟合**；
- RMSE 约 3.68 mpg，考虑到目标变量 MPG 取值范围约为 9~47 mpg，该误差水平在可接受范围内；
- 模型对高 MPG 车型（燃油效率较高的轻型车）的拟合效果较差，预测残差偏大。

#### 2.2.2 随机森林回归模型

| 评估集 | R² 分数 | RMSE | MAE |
|--------|---------|------|-----|
| 训练集 | 0.981 | 1.09 mpg | 0.77 mpg |
| 测试集 | 0.872 | 2.89 mpg | 2.11 mpg |

- **模型结构：** `RandomForestRegressor(n_estimators=100, random_state=42)`，未使用特征缩放；
- **预测结果：** 保存至 `2.2.2_results_rf.txt`；
- **评估报告：** 保存至 `2.2.2_report_rf.txt`。

**性能解读：**
- 训练集 R² = 0.981，测试集 R² = 0.872，两者差值约为 0.109，说明模型存在**一定程度的过拟合**，这是随机森林在小数据集上的常见现象；
- 与线性回归相比，随机森林在测试集上的 R² 提升约 8.6 个百分点（0.872 vs 0.786），RMSE 降低约 0.79 mpg（2.89 vs 3.68），整体预测精度显著优于线性回归；
- 随机森林能够捕捉特征间的非线性交互关系（如重量与马力的交互效应），因此在复杂数据结构上表现更佳。

#### 2.2.3 两模型综合对比

| 对比维度 | 线性回归 | 随机森林 |
|----------|----------|----------|
| 测试集 R² | 0.786 | **0.872** |
| 测试集 RMSE | 3.68 mpg | **2.89 mpg** |
| 测试集 MAE | 2.87 mpg | **2.11 mpg** |
| 过拟合程度 | 轻微（差值 0.026） | 中等（差值 0.109） |
| 模型可解释性 | **高**（系数可直接解读） | 较低（黑盒模型） |
| 训练速度 | **快** | 较慢 |
| 对异常值的鲁棒性 | 较弱 | **较强** |

**结论：** 随机森林模型在预测精度上全面优于线性回归模型，推荐作为燃油效率预测的主模型。

### 2.3 残差分析

由于本任务为回归问题，以下以残差分析替代分类任务中的混淆矩阵分析。

**残差分布特征（基于测试集）：**

| 统计量 | 线性回归 | 随机森林 |
|--------|----------|----------|
| 残差均值 | -0.12 mpg | 0.08 mpg |
| 残差标准差 | 3.68 mpg | 2.89 mpg |
| 最大正残差（低估） | +9.2 mpg | +6.8 mpg |
| 最大负残差（高估） | -8.7 mpg | -5.9 mpg |

**残差分析发现：**

1. **线性回归残差** 呈现轻微的非线性趋势，在较高 MPG 值区间（>35 mpg）残差系统性偏大，说明线性假设在极端值区间不成立；
2. **随机森林残差** 分布更为均匀，无明显系统性偏差，但在极端值（MPG < 12 或 MPG > 40）处仍存在较大误差；
3. 两模型的残差均值接近 0，说明模型整体无明显系统性偏差。

## 三、错误案例分析

### 3.1 错误类型分析

本回归任务中，"误差"定义为预测值与真实值之差（残差）。根据误差分布，将预测误差分为以下三类：

| 误差类型 | 定义 | 占比（随机森林） | 占比（线性回归） |
|----------|------|-----------------|-----------------|
| 小误差（\|残差\| ≤ 2 mpg） | 预测精度高，误差在合理范围内 | 约 58% | 约 44% |
| 中等误差（2 < \|残差\| ≤ 5 mpg） | 存在一定预测偏差，可接受 | 约 33% | 约 40% |
| 大误差（\|残差\| > 5 mpg） | 预测严重偏离，需重点关注 | 约 9% | 约 16% |

### 3.2 错误原因分析

#### 3.2.1 数据层面原因

1. **样本量不足：** 有效样本仅 392 条，对于涵盖 7 个特征的回归模型而言，训练数据相对稀少，容易导致模型在边缘区域（极高或极低 MPG）缺乏足够的学习样本；

2. **特征代表性不均衡：** 数据集中美国车型（origin=1）占比约 62%，而欧洲（origin=2）和日本（origin=3）车型合计仅占 38%。模型对美国大排量低效率车型的预测能力较强，对小排量高效率进口车的预测能力相对较弱；

3. **`horsepower` 特征缺失处理的影响：** 原始数据中 `horsepower` 含 6 个缺失值（`"?"`），直接删除对应样本可能导致特定类型车辆（通常为历史较早的车型）信息丢失；

4. **时间效应未充分捕捉：** `model year` 特征以整数编码（70–82），实际上汽车技术在此期间发生了显著变化（如 1973 年和 1979 年两次石油危机导致燃油效率要求大幅提升），简单的数值编码难以完整捕捉该非线性时间趋势。

#### 3.2.2 模型层面原因

1. **线性回归的模型假设局限：** 线性回归假设特征与目标变量之间存在线性关系，但 MPG 与车重、排量等特征之间实际呈现明显的非线性关系（如重量对 MPG 的影响具有递减边际效应）；

2. **随机森林的过拟合倾向：** 训练集 R² = 0.981 而测试集 R² = 0.872，差值达 0.109，表明在小样本数据集上随机森林存在过拟合现象，模型记住了训练样本的局部特征；

3. **随机森林对极端值预测能力不足：** 树模型的预测值受限于训练集中观测到的目标变量范围，对于训练集中少见的极端 MPG 值（如 >40 mpg），模型倾向于向均值方向收缩预测结果。

## 四、改进建议

### 4.1 数据层面改进

#### 4.1.1 增加样本量与数据多样性

- **扩充数据集：** 当前数据集仅含 392 条有效样本，建议引入更多车型数据，目标样本量提升至 2000 条以上，以增强模型的泛化能力；
- **补充近年数据：** Auto MPG 数据集覆盖年份为 1970–1982 年，汽车技术已发生根本变化。建议引入现代车型数据（混合动力、涡轮增压等），使模型适应当代智慧交通场景；
- **补充稀缺类别样本：** 针对极低 MPG（< 12）和极高 MPG（> 40）的样本不足问题，专项收集此类边缘案例数据。

#### 4.1.2 缺失值处理优化

- 当前做法直接删除含缺失值的行。建议改用**均值/中位数填充**或**KNN 插补**（`sklearn.impute.KNNImputer`）来保留更多样本，避免数据损失；
- 对于 `horsepower` 的 6 个缺失值，可通过排量、气缸数等相关特征进行回归插补，提高数据利用率。

#### 4.1.3 特征工程增强

- **添加交互特征：** 构造 `weight × displacement`、`horsepower / weight`（功重比）等交互项，显式捕捉特征间的乘法关系；
- **`model year` 特征细化：** 将年份编码为分段哑变量或使用石油危机事件标记（如 1973 年前/后、1979 年前/后），更精确地捕捉时代效应；
- **`origin` 特征编码优化：** 将产地（1/2/3）转换为 One-Hot 编码，避免模型误判产地间存在序数关系。

> **注意：** 本任务为回归问题，不存在类别不平衡问题，SMOTE（Synthetic Minority Over-sampling Technique）仅适用于分类任务，不适用于本场景。

### 4.2 模型层面改进

#### 4.2.1 随机森林超参数调优

针对当前随机森林模型的过拟合问题，建议通过以下超参数调整加以缓解：

| 超参数 | 当前值 | 建议调整方向 | 说明 |
|--------|--------|-------------|------|
| `n_estimators` | 100 | 适当增加（150~300） | 更多树可降低方差，减少过拟合 |
| `max_depth` | 无限制（None） | 设置上限（如 8~15） | 限制树深度，防止单棵树过度拟合 |
| `min_samples_split` | 2 | 提高（如 5~10） | 减少分裂次数，提高泛化能力 |
| `min_samples_leaf` | 1 | 提高（如 3~5） | 保证叶节点有足够样本 |
| `max_features` | `"auto"` | 尝试 `"sqrt"` 或 0.7 | 调整每棵树的特征采样比例 |

建议使用 `GridSearchCV` 或 `RandomizedSearchCV` 进行系统化超参数搜索：

```python
from sklearn.model_selection import GridSearchCV

param_grid = {
    'n_estimators': [100, 200, 300],
    'max_depth': [8, 12, 15, None],
    'min_samples_split': [2, 5, 10],
    'min_samples_leaf': [1, 3, 5]
}

grid_search = GridSearchCV(
    RandomForestRegressor(random_state=42),
    param_grid,
    cv=5,
    scoring='r2',
    n_jobs=-1
)
grid_search.fit(X_train, y_train)
```

#### 4.2.2 尝试 XGBoost 等梯度提升集成方法

梯度提升模型（如 XGBoost、LightGBM）在结构化数据回归任务上通常优于随机森林，原因如下：

- 采用**残差提升**策略，每棵新树专注于修正前一棵树的预测误差，收敛更高效；
- 内置**正则化项**（L1/L2），天然具备抗过拟合能力；
- 支持对**缺失值的原生处理**，无需额外预处理。

示例代码：

```python
from xgboost import XGBRegressor

xgb_model = XGBRegressor(
    n_estimators=200,
    learning_rate=0.05,
    max_depth=6,
    subsample=0.8,
    colsample_bytree=0.8,
    random_state=42
)
xgb_model.fit(X_train, y_train)
print("XGBoost 测试集 R²:", xgb_model.score(X_test, y_test))
```

#### 4.2.3 交叉验证替代单次训练测试划分

当前仅使用单次 80/20 分割，结果受随机种子影响较大。建议改用 **5 折交叉验证**（K-Fold Cross Validation），获得更稳健的性能估计：

```python
from sklearn.model_selection import cross_val_score

cv_scores = cross_val_score(rf_model, X, y, cv=5, scoring='r2')
print(f"5折交叉验证 R² 均值: {cv_scores.mean():.4f} ± {cv_scores.std():.4f}")
```

## 五、测试结论

1. **模型有效性：** 两种模型均成功完成了燃油效率回归预测任务，随机森林模型的综合性能（测试集 R² = 0.872）显著优于线性回归模型（测试集 R² = 0.786），具备实际部署价值。

2. **主要问题：** 随机森林存在一定过拟合（训练/测试 R² 差值 0.109）；两种模型对极端 MPG 值（< 12 或 > 40）的预测误差偏大；线性回归在非线性区间拟合不足。

3. **改进优先级：** 建议优先进行随机森林超参数调优（限制树深度）和特征工程增强（交互特征、One-Hot 编码产地），预期可在不增加数据量的情况下获得显著性能提升。

4. **落地建议：** 在智慧交通实际应用中，推荐以调优后的随机森林或 XGBoost 作为生产模型，线性回归可作为基准模型保留用于性能对比和可解释性分析。
