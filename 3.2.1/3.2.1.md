# 3.2.1 图像识别评估系统交互流程设计

## 1. 系统概述

### 1.1 系统功能描述

本系统基于预训练的 ResNet 深度卷积神经网络（`resnet.onnx`），实现对输入图像的自动识别与分类评估。系统核心功能如下：

- **图像输入**：支持用户上传单张 RGB 图像文件（如 `img_test.jpg`）；
- **自动预处理**：对输入图像执行标准化的预处理流程，确保与模型输入规格匹配；
- **模型推理**：调用 ONNX Runtime 推理引擎加载并运行 `resnet.onnx` 模型，完成前向推理；
- **结果输出**：输出 Top-5 预测类别及其对应的 Softmax 概率值，类别标签来源于 `labels.txt`（ImageNet 分类标签集）；
- **结果反馈**：提供可视化展示与交互反馈机制，辅助用户理解识别结果。

### 1.2 技术架构

系统推理流程采用以下标准化管道：

```
加载模型（resnet.onnx）
    │
    ▼
加载标签（labels.txt，逐行读取，共 1000 类 ImageNet 标签）
    │
    ▼
加载图片（PIL.Image.open，RGB 格式读取 img_test.jpg）
    │
    ▼
图像预处理
    ├── Resize：双线性插值缩放至 256×256
    ├── Center Crop：中心裁剪至 224×224
    ├── 归一化：像素值除以 255，映射至 [0, 1]
    ├── 标准化：均值 [0.485, 0.456, 0.406]，标准差 [0.229, 0.224, 0.225]
    ├── 转置：HWC → CHW（np.transpose）
    └── 扩展批次维度：reshape 为 (1, C, H, W)，数据类型 float32
    │
    ▼
模型推理（ort.InferenceSession.run）
    │
    ▼
概率计算（scipy.special.softmax，沿 axis=-1）
    │
    ▼
Top-5 结果提取（np.argsort 降序排列，取前 5 个类别索引）
    │
    ▼
输出 Top-5 类别名称及对应概率
```

## 2. 最优人机交互方式设计

### 2.1 交互界面设计

本系统推荐采用 **Web 图形交互界面**作为最优人机交互方式。理由如下：

| 交互方式 | 可访问性 | 易用性 | 部署复杂度 | 推荐程度 |
|----------|----------|--------|------------|----------|
| Web 界面 | 高（浏览器访问，无需安装） | 高（图形化操作） | 中 | ★★★★★ |
| 桌面应用 | 中（需安装） | 高 | 高 | ★★★ |
| 命令行界面 | 低（需技术背景） | 低 | 低 | ★★ |
| REST API 接口 | 高（程序集成） | 中（需编程） | 低 | ★★★★ |

**推荐方案：Web 图形界面（基于 Flask/Gradio/Streamlit 框架）**

Web 界面具备以下优势：
- 无需用户安装额外软件，通过浏览器即可访问；
- 支持直观的拖拽上传与可视化结果展示；
- 易于集成进现有评估平台或教学系统；
- 响应式设计可兼容桌面与移动端设备。

### 2.2 用户操作流程

#### Step 1：用户上传图片

**操作描述：**
1. 用户通过浏览器访问图像识别评估系统主页；
2. 系统呈现图像上传区域，支持以下两种方式：
   - **拖拽上传**：将本地图片文件（如 `img_test.jpg`）拖入指定区域；
   - **点击上传**：点击"选择文件"按钮，通过文件管理器选取图片；
3. 系统对上传文件进行初步校验：
   - 文件格式检查（支持 JPG、JPEG、PNG、BMP 等常见图像格式）；
   - 文件大小限制（建议设置上限为 10MB）；
   - 图像有效性验证（确保文件可被正常解码为 RGB 图像）；
4. 校验通过后，界面显示上传图片的缩略图预览，确认无误后用户点击"开始识别"按钮。

**界面元素：**
- 图片上传拖拽区域（带虚线边框，含上传图标与提示文字）
- 上传进度条
- 图片预览窗口（展示原始图片）
- "开始识别"提交按钮
- 文件格式与大小提示说明

#### Step 2：系统自动预处理

**操作描述：**

用户提交图片后，系统在后端自动执行完整预处理流程，无需用户手动干预。界面同步展示处理进度状态：

| 处理步骤 | 技术细节 | 状态提示 |
|----------|----------|----------|
| 图像加载 | `PIL.Image.open()` 读取为 RGB 格式 | "正在加载图像..." |
| 尺寸缩放 | 双线性插值，Resize 至 256×256 | "正在调整图像尺寸..." |
| 中心裁剪 | 从 256×256 中心裁剪 224×224 区域 | "正在裁剪图像..." |
| 像素归一化 | 除以 255，映射至 [0, 1] | "正在归一化像素值..." |
| 标准化处理 | 减均值除标准差（ImageNet 统计量） | "正在标准化图像..." |
| 维度转换 | HWC → CHW，添加批次维度，转 float32 | "正在准备模型输入..." |

整个预处理过程对用户透明，界面以进度动画形式呈现，预计耗时不超过 1 秒。

#### Step 3：模型推理与结果展示

**操作描述：**

预处理完成后，系统自动调用 ONNX Runtime 推理引擎执行模型推理：

1. **推理执行**：`ort.InferenceSession.run()` 将预处理后的张量送入 `resnet.onnx` 模型进行前向计算；
2. **概率计算**：对模型输出的 logits 应用 Softmax 函数（`scipy.special.softmax(output, axis=-1)`），将原始分数转换为概率分布；
3. **Top-5 提取**：使用 `np.argsort` 对概率数组降序排列，取概率最高的前 5 个类别索引，从 `labels.txt` 中检索对应标签名称；
4. **结果展示**：界面以结构化形式呈现识别结果。

**结果展示界面设计：**

#### Step 4：结果反馈与优化

**操作描述：**

结果展示后，系统提供以下交互反馈机制：

1. **结果确认**：用户可对识别结果进行主观评估，标注识别是否正确（"正确" / "错误"），辅助系统积累评估数据；
2. **结果导出**：支持将识别结果导出为 JSON 格式或 CSV 报表，便于批量评估分析；
3. **重新识别**：提供"重新上传"按钮，允许用户上传新图片进行新一轮识别；

5. **预处理参数调整（高级模式）**：允许高级用户自定义预处理参数（如裁剪策略、归一化均值方差），观察不同预处理设置对识别结果的影响。

## 3. 交互优化建议

### 3.1 性能优化

**批量处理机制**

针对需要批量评估多张图片的场景，设计批量上传功能：
- 支持一次上传多张图片（建议单批次不超过 32 张）；
- 后端将图片组装为批量张量（batch_size > 1），利用 GPU/CPU 并行计算提升吞吐量；
- 前端以进度条形式展示批量处理进度，实时更新已完成数量。

**模型缓存机制**

- **会话级缓存**：`ort.InferenceSession` 在服务启动时仅初始化一次，避免每次请求重复加载 `resnet.onnx` 模型文件，显著降低延迟；
- **标签缓存**：`labels.txt` 内容在服务启动时一次性读入内存，以列表形式缓存，避免重复 I/O；
- **推理结果缓存**：对相同 MD5 哈希值的图片，可直接返回已缓存的推理结果，避免重复计算（适用于演示与测试场景）。

**异步推理**

采用异步任务队列（如 Celery + Redis）处理推理请求，避免长时间推理阻塞主线程，提升系统并发处理能力。

### 3.2 用户体验优化

- **实时预览**：图片上传完成后立即在界面显示预览缩略图，让用户在等待推理的同时确认上传内容；
- **进度反馈**：推理过程中显示动态加载动画（骨架屏或进度条），消除用户等待焦虑；
- **结果高亮**：Top-1 预测结果以更大字体和鲜明颜色突出显示，便于用户快速定位最优结果；
- **置信度提示**：当 Top-1 概率低于 50% 时，在结果旁增加"置信度较低，识别结果仅供参考"的提示；
- **移动端适配**：采用响应式布局，确保在手机浏览器上同样具备良好的上传与结果查看体验；
- **无障碍设计**：为图片预览和识别结果添加 alt 文本及 ARIA 属性，兼顾视觉辅助需求。

### 3.3 可靠性保障

- **输入合法性验证**：在前端和后端双重校验上传图像的格式、尺寸和完整性，防止异常输入导致推理崩溃；
- **异常捕获与日志记录**：后端对推理全流程（加载、预处理、推理、后处理）使用 try-except 捕获异常，并写入结构化日志，便于问题排查；
- **服务健康检查**：提供 `/health` 接口，定期检测 ONNX 推理引擎及模型加载状态，确保服务正常运行；
- **并发限制**：设置合理的并发推理请求上限，防止资源耗尽导致服务不可用；
- **数据隐私保护**：用户上传的图片仅用于本次推理，处理完成后及时从服务器删除，不留存原始图像数据。

## 4. 实施建议

### 4.1 技术选型建议

| 组件 | 推荐技术 | 说明 |
|------|----------|------|
| 后端框架 | Flask 或 FastAPI | 轻量级，易于集成 ONNX Runtime |
| 推理引擎 | onnxruntime==1.x | 跨平台，支持 CPU/GPU 推理 |
| 前端框架 | Gradio 或 Streamlit | 专为 AI 展示设计，极低开发成本 |
| 图像处理 | Pillow (PIL) + NumPy | 与现有预处理代码完全兼容 |
| 概率计算 | scipy.special.softmax | 与现有推理代码一致 |
| 部署方式 | Docker 容器化部署 | 环境隔离，便于迁移与扩展 |

### 4.2 快速原型方案

针对教学评估场景，推荐使用 **Gradio** 快速搭建交互界面，核心代码结构如下：

```python
import gradio as gr
import onnxruntime as ort
import numpy as np
from PIL import Image
from scipy.special import softmax

# 初始化：加载模型与标签（仅执行一次）
session = ort.InferenceSession("resnet.onnx")
input_name = session.get_inputs()[0].name
output_name = session.get_outputs()[0].name
with open("labels.txt", "r") as f:
    labels = [line.strip() for line in f.readlines()]

def preprocess(image: Image.Image) -> np.ndarray:
    """图像预处理：resize 256 → crop 224 → 归一化 → 标准化"""
    image = image.resize((256, 256), Image.BILINEAR)
    left = (256 - 224) // 2
    image = image.crop((left, left, left + 224, left + 224))
    img_array = np.array(image).astype(np.float32) / 255.0
    mean = np.array([0.485, 0.456, 0.406])
    std  = np.array([0.229, 0.224, 0.225])
    img_array = (img_array - mean) / std
    img_array = np.transpose(img_array, (2, 0, 1))  # HWC → CHW
    img_array = img_array.reshape(1, 3, 224, 224)   # 添加批次维度
    return img_array.astype(np.float32)

def predict(image: Image.Image) -> dict:
    """推理并返回 Top-5 结果"""
    tensor = preprocess(image)
    output = session.run([output_name], {input_name: tensor})[0]
    probs = softmax(output, axis=-1)[0]
    top5_idx = np.argsort(probs)[::-1][:5]
    return {labels[i]: float(probs[i]) for i in top5_idx}

# 构建 Gradio Web 界面
demo = gr.Interface(
    fn=predict,
    inputs=gr.Image(type="pil", label="上传待识别图片"),
    outputs=gr.Label(num_top_classes=5, label="Top-5 识别结果"),
    title="图像识别评估系统（ResNet / ONNX）",
    description="上传一张图片，系统将自动完成预处理与推理，输出 Top-5 类别及概率。"
)

demo.launch(server_name="0.0.0.0", server_port=7860)
```

### 4.3 实施步骤

1. **环境准备**：安装 `onnxruntime`、`Pillow`、`numpy`、`scipy`、`gradio` 依赖包；
2. **资源就位**：确认 `resnet.onnx`、`labels.txt`、`img_test.jpg` 位于同一工作目录；
3. **本地测试**：运行脚本，通过浏览器访问 `http://localhost:7860` 验证功能；

5. **部署上线**：如需多用户访问，将服务打包为 Docker 镜像并部署至服务器；
6. **监控维护**：配置日志收集与告警机制，持续监控推理延迟与错误率。

## 总结

本方案以 **Web 图形界面**作为图像识别评估系统的最优人机交互方式，基于 `resnet.onnx` 模型的标准推理管道（加载模型 → 加载标签 → 加载图片 → 图像预处理 → 模型推理 → 输出 Top-5 概率类别），设计了完整的四步用户操作流程。通过直观的拖拽上传、实时进度反馈、可视化结果展示以及用户反馈收集机制，最大程度降低用户操作门槛，同时借助模型缓存、批量推理、异步处理等优化手段保障系统性能与可靠性。推荐在教学评估场景中优先采用 Gradio 框架快速搭建原型，在生产环境中进一步升级为 FastAPI + 前端分离的架构以满足更高并发需求。
