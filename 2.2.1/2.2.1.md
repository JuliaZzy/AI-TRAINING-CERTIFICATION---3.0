## 二、模型性能评估

### 2.1 评估指标说明

本次评估采用以下指标体系，综合考量模型在不平衡数据场景下的表现：

- **准确率（Accuracy）：** 所有样本中预测正确的比例。在类别不平衡场景下，该指标可能产生误导，需结合其他指标综合判断。
- **精确率（Precision）：** 模型预测为正例（拖欠）中真实拖欠的比例，衡量预测的"准"。
- **召回率（Recall）：** 真实拖欠样本中被正确预测出来的比例，衡量预测的"全"，对信用风险场景尤为重要。
- **F1 分数（F1-Score）：** 精确率与召回率的调和平均，综合评估模型对少数类的识别能力。
- **混淆矩阵：** 直观展示真正例（TP）、假正例（FP）、真负例（TN）、假负例（FN）的分布。

### 2.2 模型一：原始不平衡数据训练结果

使用原始训练集（未做过采样处理）训练 Logistic 回归模型，测试集评估结果如下：

**分类报告（2.2.1_results.txt）：**

```
              precision    recall  f1-score   support

           0       0.94      0.99      0.96     27984
           1       0.62      0.23      0.34      2016

    accuracy                           0.93     30000
   macro avg       0.78      0.61      0.65     30000
weighted avg       0.92      0.93      0.92     30000
```

**混淆矩阵（模型一）：**

|  | 预测：正常（0） | 预测：拖欠（1） |
|---|---|---|
| **实际：正常（0）** | TN = 27,704 | FP = 280 |
| **实际：拖欠（1）** | FN = 1,552 | TP = 464 |

**结果分析：**

- 整体准确率为 **93.3%**，但该数值因类别不平衡而虚高；
- 对多数类（正常还款，类别 0）的预测表现优异，召回率达 **99%**；
- 对少数类（严重拖欠，类别 1）的召回率仅为 **23%**，即约 77% 的真实拖欠客户未被识别，存在显著的漏报风险；
- 少数类 F1 分数仅为 **0.34**，表明模型对拖欠风险的识别能力严重不足。

### 2.3 模型二：SMOTE 过采样后训练结果

对训练集应用 SMOTE 过采样（`random_state=42`）后重新训练 Logistic 回归模型，测试集评估结果如下：

**分类报告（2.2.1_results_xg.txt）：**

```
              precision    recall  f1-score   support

           0       0.97      0.74      0.84     27984
           1       0.24      0.78      0.37      2016

    accuracy                           0.74     30000
   macro avg       0.60      0.76      0.60     30000
weighted avg       0.91      0.74      0.80     30000
```

**混淆矩阵（模型二）：**

|  | 预测：正常（0） | 预测：拖欠（1） |
|---|---|---|
| **实际：正常（0）** | TN = 20,708 | FP = 7,276 |
| **实际：拖欠（1）** | FN = 443 | TP = 1,573 |

**结果分析：**

- 整体准确率下降至 **74%**，但这是以牺牲多数类预测精度换取少数类召回率提升的合理权衡；
- 少数类（拖欠，类别 1）的召回率大幅提升至 **78%**，意味着约 78% 的真实拖欠客户被成功识别；
- 少数类 F1 分数提升至 **0.37**，少数类识别能力有所改善；
- 代价是多数类预测精度有所下降，误报率（FP）上升，需结合业务场景的误报与漏报成本进行权衡。

### 2.4 两轮模型性能对比汇总

| 评估指标 | 模型一（原始数据） | 模型二（SMOTE 过采样） |
|----------|:-----------------:|:---------------------:|
| 整体准确率（Accuracy） | 93.3% | 74.0% |
| 类别1 精确率（Precision） | 62% | 24% |
| 类别1 召回率（Recall） | 23% | 78% |
| 类别1 F1 分数 | 0.34 | 0.37 |
| 宏平均 F1（Macro F1） | 0.65 | 0.60 |
| 假负例数（FN，漏报） | 1,552 | 443 |
| 假正例数（FP，误报） | 280 | 7,276 |

> **综合结论：** 在信用风险场景中，漏报（将拖欠客户判断为正常）的代价通常远高于误报。因此，SMOTE 处理后的模型二在业务价值层面更优，召回率从 23% 提升至 78% 是实质性改进。

## 三、错误案例分析

### 3.1 错误类型分析

本模型的预测误差主要分为两类：

**（1）假负例（False Negative，FN）—— 漏报风险**

- **定义：** 实际发生严重拖欠（标签为 1），但模型预测为正常（标签为 0）。
- **业务影响：** 高风险借款人被误判为低风险，可能导致信贷机构遭受实际违约损失，是信用评分场景中危害最大的错误类型。
- **模型一中数量：** 约 1,552 例（占全部拖欠样本的 77%）。

**（2）假正例（False Positive，FP）—— 误报风险**

- **定义：** 实际正常还款（标签为 0），但模型预测为拖欠（标签为 1）。
- **业务影响：** 正常客户被错误拒贷，影响业务量与客户体验，但财务损失相对可控。
- **模型二中数量：** 约 7,276 例（SMOTE 后召回率提升的代价）。

### 3.2 错误原因分析

**（1）数据层面原因**

- **严重类别不平衡：** 数据集中拖欠样本仅占约 6.7%，模型在训练时倾向于将所有样本预测为多数类，导致少数类识别能力极弱。
- **特征缺失值：** `MonthlyIncome` 与 `NumberOfDependents` 字段存在较多缺失值，简单填充策略（如均值填充）可能引入噪声，影响模型对边界样本的判断。
- **异常值干扰：** `RevolvingUtilizationOfUnsecuredLines` 存在超过 1 的异常值（信贷使用率理论上不超过 1），`DebtRatio` 同样存在极端异常值，未经充分清洗会影响线性决策边界。

**（2）模型层面原因**

- **线性假设局限：** Logistic 回归假设特征与对数几率之间存在线性关系，但信用违约行为受多个特征的非线性交互影响，线性模型的表达能力有限。
- **决策阈值默认设定：** 默认使用 0.5 作为分类阈值，在不平衡数据下不是最优选择，调低阈值（如 0.3）可在不重新训练的前提下提高少数类召回率。
- **特征共线性：** 多个逾期相关特征（如 30-59 天、60-89 天、90 天以上逾期次数）之间存在较强相关性，可能影响回归系数的稳定性。

### 3.3 具体错误案例说明

以下展示三类典型误判案例（数值为示例性描述）：

**案例一：边界拖欠客户被漏报（FN）**

| 特征 | 数值 |
|------|------|
| `RevolvingUtilizationOfUnsecuredLines` | 0.45 |
| `age` | 52 |
| `NumberOfTime30-59DaysPastDueNotWorse` | 1 |
| `DebtRatio` | 0.38 |
| `MonthlyIncome` | 缺失（填充为均值 6,500） |
| **实际标签** | 1（拖欠） |
| **模型预测** | 0（正常） |

**分析：** 该客户仅有一次轻微逾期记录，收入数据缺失被均值填充后掩盖了真实财务压力，信贷使用率处于中等水平，各项特征均未触发模型的拖欠判断阈值，导致漏报。

**案例二：高逾期频率客户被正确识别（TP）**

| 特征 | 数值 |
|------|------|
| `NumberOfTimes90DaysLate` | 5 |
| `NumberOfTime30-59DaysPastDueNotWorse` | 8 |
| `RevolvingUtilizationOfUnsecuredLines` | 0.92 |
| **实际标签** | 1（拖欠） |
| **模型预测** | 1（拖欠） |

**分析：** 多次严重逾期记录叠加高信贷使用率，特征信号明显，模型能够正确识别。

**案例三：收入不稳定客户被误报（FP）**

| 特征 | 数值 |
|------|------|
| `DebtRatio` | 1.85（异常高值） |
| `MonthlyIncome` | 1,200 |
| `NumberOfTime30-59DaysPastDueNotWorse` | 0 |
| **实际标签** | 0（正常） |
| **模型预测** | 1（拖欠） |

**分析：** 该客户债务比率异常偏高，但实际还款记录良好（逾期次数为 0）。模型主要依赖 `DebtRatio` 特征做出拖欠判断，未能充分识别该特征的异常值属性，导致误报。

## 四、改进建议

### 4.1 数据层面改进

**（1）不平衡数据处理**

- **已实施（SMOTE）：** 已通过 SMOTE 对少数类进行合成过采样，少数类召回率从 23% 提升至 78%，效果显著。
- **进一步优化方向：**
  - 尝试 **ADASYN（自适应合成采样）**，对难以分类的边界样本赋予更高采样权重；
  - 结合 **欠采样（如 RandomUnderSampler 或 Tomek Links）** 与 SMOTE 组成混合策略，在提升召回率的同时控制误报率；
  - 调整 Logistic 回归的 `class_weight='balanced'` 参数，无需重采样即可一定程度缓解不平衡问题。

**（2）缺失值处理优化**

- 对 `MonthlyIncome` 建议使用基于其他特征的 **多重插补（Multiple Imputation）** 或 **KNN 插补**，而非简单均值填充；
- 对 `NumberOfDependents` 可结合业务逻辑，将缺失值作为独立类别处理或使用众数填充。

**（3）异常值清洗**

- 对 `RevolvingUtilizationOfUnsecuredLines` 大于 1 的值进行截断或删除，还原特征的实际业务含义；
- 对 `DebtRatio` 应用分位数裁剪（如 99% 分位数截断），降低极端异常值对决策边界的干扰。

**（4）增加样本量**

- 在数据隐私合规前提下，引入更多历史借贷数据，尤其是丰富少数类（真实拖欠）样本，从根本上改善类别不平衡问题。

### 4.2 模型层面改进

**（1）超参数调优**

- 对 Logistic 回归的正则化系数 `C` 进行网格搜索（Grid Search）或随机搜索（Random Search）；
- 尝试 `penalty='l1'`（Lasso 正则化），实现稀疏化特征选择，自动过滤低贡献特征；
- 使用 `solver='saga'` 以支持 L1 正则化与大规模数据训练。

**（2）决策阈值优化**

- 绘制 **ROC 曲线** 和 **Precision-Recall 曲线**，选择最优分类阈值（如通过最大化 F1 分数或 Youden 指数）；
- 在信用风险业务中，通常将阈值调低至 0.2~0.3，优先保证对拖欠客户的识别率。

**（3）特征工程**

- 构造交互特征，如 `逾期次数合计 = 30-59天逾期 + 60-89天逾期 + 90天以上逾期`；
- 对年龄、月收入等连续型特征进行分箱（Binning），捕获非线性效应；
- 对 `DebtRatio` 与 `MonthlyIncome` 构造"绝对债务额"派生特征，提升特征的经济含义表达。

**（4）尝试更强模型**

- **随机森林（Random Forest）** 或 **梯度提升树（XGBoost / LightGBM）**：对非线性关系建模能力强，自带特征重要性分析；
- **集成学习（Ensemble）**：将 Logistic 回归与树模型进行 Stacking，提升整体泛化能力。

### 4.3 纠正后模型效果对比

| 评估指标 | 模型一（原始） | 模型二（SMOTE） | 改进方向预期效果 |
|----------|:------------:|:--------------:|:---------------:|
| 整体准确率 | 93.3% | 74.0% | 约 80~85% |
| 类别1 召回率 | 23% | 78% | 约 80~85% |
| 类别1 精确率 | 62% | 24% | 约 40~50% |
| 类别1 F1 分数 | 0.34 | 0.37 | 约 0.55~0.65 |
| 宏平均 F1 | 0.65 | 0.60 | 约 0.70~0.75 |

> **改进路径说明：** 预期通过"SMOTE + 阈值优化 + 异常值清洗 + 特征工程"的组合策略，可将类别 1 的 F1 分数提升至 0.55 以上，在保持召回率 80%+ 的同时，将精确率从 24% 提升至 40~50%，实现误报与漏报的更优平衡。

## 五、结论

本次测试系统评估了基于 GiveMeSomeCredit 数据集开发的 Logistic 回归信用评分模型。核心发现如下：

1. **原始数据训练的模型**整体准确率高达 93.3%，但在严重不平衡数据下，该指标产生误导，模型对真实拖欠客户的召回率仅 23%，不满足信用风险业务需求；

2. **SMOTE 过采样处理**有效缓解了类别不平衡问题，将拖欠客户召回率提升至 78%，是本次实验中最关键的改进措施；

3. **主要误差来源**为类别不平衡、特征缺失值与异常值处理不足，以及 Logistic 回归模型对非线性关系的表达限制；

4. **后续建议**优先落地的改进措施为：优化异常值处理、调整决策阈值、引入 `class_weight='balanced'` 参数，并尝试梯度提升树等非线性模型，以进一步提升模型的业务实用价值。
