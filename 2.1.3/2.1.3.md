# 信用评分模型数据清洗和标注流程设计

## 一、项目概述

本规范适用于信用评分模型的数据准备阶段，数据集为 `finance数据集.csv`，共包含 15000 条记录、10 个自变量和 1 个因变量（`SeriousDlqin2yrs`）。本文档从数据清洗规范与数据标注规范两个维度，对数据处理流程进行系统性说明，以保障模型训练数据的质量与可追溯性。

---

## 二、数据清洗规范

### 2.1 数据加载规范

**目的：** 确保原始数据完整、准确地载入内存，为后续处理提供可靠基础。

**操作规范：**

1. 使用 `pandas` 库的 `read_csv()` 方法加载 CSV 格式数据集。
2. 加载完成后，使用 `data.head()` 查看前5行，初步核验字段名称与数据内容是否符合预期。
3. 使用 `data.info()` 和 `data.describe()` 对数据集整体结构、字段类型及基本统计量进行审查。
4. 记录数据集原始行数与列数，作为清洗前的基准值。

**字段说明：**

| 字段名 | 含义 | 类型 |
|---|---|---|
| SeriousDlqin2yrs | 2年内严重逾期（目标变量） | int/float |
| RevolvingUtilizationOfUnsecuredLines | 无担保信用额度使用率 | float |
| age | 借款人年龄 | int |
| NumberOfTime30-59DaysPastDueNotWorse | 30-59天逾期次数 | int |
| DebtRatio | 债务比率 | float |
| MonthlyIncome | 月收入 | float |
| NumberOfOpenCreditLinesAndLoans | 开放信贷额度和贷款数量 | int |
| NumberOfTimes90DaysLate | 90天以上逾期次数 | int |
| NumberRealEstateLoansOrLines | 房地产贷款或信用额度数量 | int |
| NumberOfTime60-89DaysPastDueNotWorse | 60-89天逾期次数 | int |
| NumberOfDependents | 家庭赡养人数 | int |

---

### 2.2 缺失值处理规范

**目的：** 消除缺失数据对模型训练造成的偏差，保证数据完整性。

**操作规范：**

1. 使用 `data.isnull().sum()` 统计各字段缺失值数量，并计算缺失率。
2. 根据缺失率采用差异化处理策略：
   - **数值型字段（缺失率 < 20%）：** 采用该字段的**中位数**进行填充，以降低极端值对均值的干扰。
   - **数值型字段（缺失率 ≥ 20%）：** 评估该字段对目标变量的重要性，必要时可考虑删除该字段。
   - **类别型字段：** 采用该字段的**众数**进行填充。
3. 处理完成后，再次验证所有字段缺失值数量均为 0。
4. 记录各字段缺失值数量及填充方法，纳入数据处理日志。

---

### 2.3 重复值处理规范

**目的：** 避免重复样本导致模型过拟合，保证训练数据的独立性。

**操作规范：**

1. 使用 `data.duplicated()` 方法识别完全重复的记录行。
2. 使用 `duplicates.sum()` 统计重复行数量，并记录该数值。
3. 使用 `data[~duplicates]` 删除所有重复行，保留首次出现的记录。
4. 在终端或日志中打印删除行数，示例输出格式如下：

   ```
   删除的重复行数: <N>
   ```

5. 删除操作须在异常值处理之后执行，避免异常值清洗改变行索引引发误判。

---

### 2.4 异常值处理规范

**目的：** 识别并移除统计意义上偏离正常范围的极端值，防止其影响模型的学习效果。

**检测方法：箱线图可视化结合 IQR（四分位距）统计裁剪**

**操作规范：**

1. **可视化检测：** 使用 `seaborn.boxplot()` 对所有数值型字段绘制箱线图，直观观察各字段的数据分布及潜在离群点分布情况，图形尺寸统一设置为 `(12, 8)`。

2. **IQR 方法定义异常值边界：**

   - 计算第一四分位数：$Q_1 = \text{data.quantile}(0.25)$
   - 计算第三四分位数：$Q_3 = \text{data.quantile}(0.75)$
   - 计算四分位距：$IQR = Q_3 - Q_1$
   - 异常值下界：$Q_1 - 1.5 \times IQR$
   - 异常值上界：$Q_3 + 1.5 \times IQR$

3. **异常值移除：** 删除任意数值型字段超出上述边界的记录行，采用行级过滤策略，即某行中**任意一列**出现异常值，则整行删除。

4. 记录处理前后数据集行数，计算异常值移除比例，纳入数据处理日志。

**注意事项：** 异常值处理须在缺失值填充完成后执行；对于业务含义明确的字段（如年龄、月收入），须结合业务逻辑验证边界的合理性。

---

### 2.5 数据类型转换规范

**目的：** 确保各字段数据类型与其业务含义及后续算法要求保持一致。

**操作规范：**

1. 使用 `data.dtypes` 检查所有字段的数据类型。
2. 对于应为数值型但被读取为字符串的字段，使用 `pd.to_numeric()` 进行转换，并设置 `errors='coerce'` 将无法转换的值置为 `NaN`，随后按缺失值规范处理。
3. 目标变量 `SeriousDlqin2yrs` 须确保为整型（`int`）或浮点型（`float`），以满足分类模型的输入要求。
4. 字段类型转换须在归一化处理之前完成。

---

### 2.6 数据标准化/归一化规范

**目的：** 消除不同量纲特征之间的差异，使各特征在同一数值范围内参与模型训练，提升模型收敛速度与精度。

**方法：Min-Max 归一化（最小-最大缩放）**

**归一化公式：**

$$x' = \frac{x - x_{\min}}{x_{\max} - x_{\min}}$$

归一化后，所有数值型特征的值域均缩放至 $[0, 1]$ 区间。

**操作规范：**

1. 使用 `sklearn.preprocessing.MinMaxScaler` 对所有数值型字段（`numeric_cols`）进行归一化处理。
2. 归一化须在特征工程（新特征创建）完成之后执行，确保新特征同步纳入缩放范围。
3. 使用 `scaler.fit_transform()` 对清洗后的数据集进行拟合并转换，禁止在测试集上单独 `fit`，以防止数据泄露。
4. 保存 `scaler` 对象，供后续模型推理时对新数据进行一致性变换。

---

### 2.7 特征工程规范

**目的：** 通过构造新特征，充分挖掘原始数据中蕴含的业务信息，增强模型对信用风险的判别能力。

**操作规范：**

#### 新特征一：收入负债比（IncomeToDebtRatio）

| 项目 | 内容 |
|---|---|
| 特征名称 | `IncomeToDebtRatio` |
| 计算公式 | `MonthlyIncome / DebtRatio` |
| 业务含义 | 衡量借款人月收入相对于债务负担的能力，数值越高表明偿债能力越强，信用风险越低 |
| 创建时机 | 在缺失值处理、异常值处理完成后，归一化处理之前 |
| 注意事项 | 需防范 `DebtRatio` 为 0 的情况，可预先将 0 值替换为极小正数（如 `1e-6`），避免除零错误 |

#### 新特征二：月收入（MonthlyIncome）

| 项目 | 内容 |
|---|---|
| 特征名称 | `MonthlyIncome` |
| 业务含义 | 作为独立特征直接参与模型训练，反映借款人的基本偿还能力 |
| 处理要求 | 该字段存在缺失值，须按缺失值处理规范完成填充后再参与特征工程 |

**特征创建总体原则：**

1. 所有新特征须具备明确的业务解释性，避免引入无实际意义的组合特征。
2. 特征创建完成后，检查新特征的描述统计量（均值、方差、最大/最小值），确认无异常。
3. 可使用相关系数热力图评估新特征与目标变量的相关性，以及与原有特征的共线性。

## 三、数据标注规范

### 3.1 目标变量定义

| 项目 | 内容 |
|---|---|
| 目标变量名称 | `SeriousDlqin2yrs` |
| 含义 | 借款人在未来 2 年内是否发生严重逾期（90天及以上） |
| 取值类型 | 二分类标签 |
| 标签值说明 | `1`：发生严重逾期（正样本，高风险）；`0`：未发生严重逾期（负样本，低风险） |
| 标注来源 | 原始数据集中的已有标注，无需人工重新标注 |

**类别不均衡说明：** 信用数据中通常存在正负样本比例失衡问题（违约样本占比较小）。在模型训练前，须统计 `SeriousDlqin2yrs` 中 `1` 和 `0` 的比例，如正样本比例低于 10%，应考虑使用过采样（SMOTE）或调整类别权重等方法进行处理。

---

### 3.2 特征变量选择依据

**特征集定义：** 将目标变量 `SeriousDlqin2yrs` 列从数据集中分离，其余所有列（含新构造特征）作为特征变量参与模型训练。

**特征变量汇总：**

| 编号 | 特征名称 | 业务含义 | 选择依据 |
|---|---|---|---|
| 1 | RevolvingUtilizationOfUnsecuredLines | 无担保循环信用额度使用率 | 反映信用卡等循环信贷的使用压力，与违约强相关 |
| 2 | age | 借款人年龄 | 年龄与还款稳定性相关，是传统信用评分的重要因子 |
| 3 | NumberOfTime30-59DaysPastDueNotWorse | 30-59天逾期次数 | 历史轻度逾期行为是未来违约的预警指标 |
| 4 | DebtRatio | 债务比率 | 衡量债务占收入的比例，反映整体财务压力 |
| 5 | MonthlyIncome | 月收入 | 直接反映还款能力 |
| 6 | NumberOfOpenCreditLinesAndLoans | 开放信贷额度和贷款数量 | 多头借贷行为与信用风险具有关联性 |
| 7 | NumberOfTimes90DaysLate | 90天以上逾期次数 | 严重历史逾期是未来违约的强预测因子 |
| 8 | NumberRealEstateLoansOrLines | 房地产贷款或信用额度数量 | 反映借款人资产状况与负债结构 |
| 9 | NumberOfTime60-89DaysPastDueNotWorse | 60-89天逾期次数 | 中度历史逾期行为的量化指标 |
| 10 | NumberOfDependents | 家庭赡养人数 | 影响可支配收入，间接反映还款压力 |
| 11 | IncomeToDebtRatio | 收入负债比（新构造） | 综合衡量收入与债务的相对关系，增强模型判别力 |

---

### 3.3 数据集划分规范

**目的：** 合理划分训练集与测试集，客观评估模型的泛化能力。

**划分方案：**

| 子集 | 比例 | 用途 |
|---|---|---|
| 训练集（X_train, y_train） | 80% | 用于模型参数学习 |
| 测试集（X_test, y_test） | 20% | 用于模型性能的最终评估 |

**操作规范：**

1. 使用 `sklearn.model_selection.train_test_split()` 进行随机划分。
2. 设置 `test_size=0.2`，`random_state=42`，确保划分结果可复现。
3. 划分须在所有数据清洗和特征工程步骤完成后执行。
4. 划分完成后，打印并记录训练集与测试集的数据形状，验证比例符合预期：

   ```
   训练数据形状: (N_train, 11)
   测试数据形状: (N_test, 11)
   ```

5. 检查训练集与测试集中目标变量的类别分布，确认两者比例基本一致（分层抽样验证）。

---

### 3.4 标注质量控制

**目的：** 确保标注数据的准确性、一致性和可追溯性，为模型训练提供高质量的标签。

**质量控制措施：**

#### 3.4.1 标签一致性检查

1. 验证目标变量 `SeriousDlqin2yrs` 的取值范围，确保仅包含 `0` 和 `1` 两种值，不存在异常标签（如负数、空值或非整数值）。
2. 对异常标签记录进行人工复核，确认后方可删除或修正。

#### 3.4.2 类别分布审查

1. 统计正负样本数量及比例，形成类别分布报告。
2. 如数据集存在严重类别不均衡（正样本比例 < 10%），须在模型训练阶段采取相应的处理措施，并在报告中注明。

#### 3.4.3 数据版本管理

1. 原始数据集 `finance数据集.csv` 须备份保存，禁止对原始文件进行覆盖修改。
2. 清洗后的数据集保存为独立文件 `2.1.3_cleaned_data.csv`，记录清洗时间、处理步骤及关键统计指标。
3. 数据处理脚本（`2.1.3.ipynb`）须纳入版本控制，确保处理过程可复现。

#### 3.4.4 处理过程日志

每次数据清洗完成后，须记录以下信息，保存至数据处理日志：

## 四、数据处理流程总览

```
原始数据加载 (finance数据集.csv, 15000条)
        |
        v
缺失值检测与填充 (数值型字段使用中位数填充)
        |
        v
异常值检测与处理 (箱线图可视化 + IQR方法，行级删除)
        |
        v
重复值检测与删除 (记录删除行数)
        |
        v
数据类型校验与转换
        |
        v
特征工程 (构造 IncomeToDebtRatio 新特征)
        |
        v
数据归一化 (Min-Max Scaler，值域缩放至[0,1])
        |
        v
目标变量分离 (SeriousDlqin2yrs 为 y，其余为 X)
        |
        v
数据集划分 (训练集80% / 测试集20%，random_state=42)
        |
        v
清洗数据保存 (2.1.3_cleaned_data.csv)
```
